| ビューポート / シナリオ | 結果 |

| 390×844 / 会話・作業タブ・下部ナビゲーション | 通過。1カラムで会話と`work.started`を切替表示し、下部ナビゲーションとの重なりはなかった。 |

## 検証中に行った改善

- Viteの`/api`開発プロキシでWebSocket中継を有効化した。修正前はHTTP healthが通っていてもUIのイベント接続が`再接続待ち`になった。修正後は`接続中`となり、`work.started`を受信できた。

## 実装した表示構造

- スマートフォン: 会話、作業、差分、ファイル、実行の下部ナビゲーションと1カラム表示。
- タブレット: workspace railと作業表示を2列にする。
- PC: workspace rail、中央作業領域、状態パネルの3領域にする。
- 全端末: 接続状態、Active work、読み取り専用ラベル、開始不可の理由を文字列でも表示する。
